# Docs

Documentation lives here. Intentionally empty at bootstrap — add ADRs,
runbooks, schema docs, and architecture notes as features land.

Suggested structure as things grow:

```
docs/
  adr/        # Architecture decision records (one per decision)
  runbooks/   # Operational procedures (deploys, incident response, migrations)
  schema/     # Database schema docs, diagrams, rationale
  api/        # API reference material (OpenAPI, auth flows)
```
